## About the API

OpenNG groups public data under resource routes like `/v1/holidays`. Most integrations follow the same pattern: discover what exists, inspect a resource `meta` endpoint, then request the dataset itself.

Production requests go to `https://api.openng.dev`. Local development usually runs at `http://localhost:3000`. Use `GET /meta` to inspect available resources and versions before wiring a client, and use `GET /health` for readiness checks behind load balancers, monitors, and CI.

<Callout title="Good first request">

If you are exploring for the first time, start with `/meta`, then check `/v1/holidays/meta`, then make a real dataset request. That sequence tells you what exists before you hard-code filters.

</Callout>

### Versioning

Paths are grouped as `/v1/{resource}`. Adding fields or filters is non-breaking. Incompatible changes to a specific resource earn a new `/v2/{resource}` only when needed.

## Create an API key

If you are building a script, backend integration, or worker, use an API key instead of browser cookies.

<Callout title="Store it once">

After you create a key in the dashboard, copy it immediately into a password manager, secret store, or environment variable. The plaintext token is shown once.

</Callout>

### Dashboard flow

1. Sign in on the main OpenNG app with your email.
2. Open `Dashboard -> API keys`.
3. Create a key and save the returned token.
4. Send it as a bearer token on API requests.

  <CodeBlockTabsList>

export OPENNG_API_KEY="ong_live_your_key_here"

curl -sS "https://api.openng.dev/v1/holidays?limit=5" \

```

const response = await fetch("https://api.openng.dev/v1/holidays?limit=5", {
  headers: {
    Authorization: `Bearer ${process.env.OPENNG_API_KEY}`,
  },
});

const json = await response.json();
```

### Rotation

Revoke the current key, then create a new one. Old keys stop working immediately once revoked.

## Terminology

**Resource**: A dataset served at `/v1/{name}` with list, `meta`, and detail routes generated from one config.

**Envelope**: The JSON response shape with `success`, optional `data` or `error`, and `meta`.

**Staging**: Import tables where rows are validated before promotion to `public`.

**Batch**: The identifier tying imported rows together for validation and migration jobs.

## FAQ

<Accordions className="getting-started-faq" type="single">
  <Accordion title="Do I need an API key?" id="do-i-need-an-api-key">
    No for basic use. Keys raise rate limits and unlock higher-throughput access for scripts and integrations.
  </Accordion>
  <Accordion title="Is there GraphQL?" id="is-there-graphql">
    Not currently. OpenNG is a REST API under `/v1`.
  </Accordion>
  <Accordion title="Where is the OpenAPI spec?" id="where-is-the-openapi-spec">
    It will be published with the docs as generation matures. Until then, use `/meta` and each resource's `/meta` route for machine-readable discovery.
  </Accordion>
  <Accordion title="How do I report bad data?" id="how-do-i-report-bad-data">
    See [Contributing](/contributing) and include the official source, the affected endpoint, and the identifiers you received.
  </Accordion>
</Accordions>
