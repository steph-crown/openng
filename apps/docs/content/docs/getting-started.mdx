OpenNG exposes **versioned JSON** under `https://api.openng.dev` (production). Local development typically uses `http://localhost:3000` when you run the API from this monorepo.

## About the API

### Versioning

Paths are grouped as `/v1/{resource}`. Adding fields or filters is non-breaking; incompatible changes to a resource earn a new `/v2/{resource}` when required.

### Discovery

- `GET /meta` lists resources and supported versions.
- `GET /v1/{resource}/meta` returns filters, columns, and dataset metadata for that resource.

### Health

`GET /health` reports database and Redis checks—use it behind load balancers and in CI.

## Create an API key

### Web dashboard

Sign in on the main OpenNG app with your email. Open **Dashboard → API keys** and create a key. The full token is shown **once**—store it in a password manager or secret store.

### Format

Keys use the prefix `ong_live_` followed by a high-entropy segment. Send the entire string as:

```
Authorization: Bearer ong_live_…
```

### Rotation

Revoke the current key, then create a new one. Old keys stop working immediately once revoked.

## Authentication

### Browser sessions

The marketing and dashboard apps use **passwordless email**. Links are single-use and short-lived; successful verification sets an **HttpOnly** session cookie.

### API keys

Server-to-server and script access should use **API keys** rather than scraping cookies. Keys are hashed in the database; plaintext never persists after creation.

### Anonymous access

Public `GET` endpoints work without a key at lower rate limits—ideal for experiments and prototypes.

## Terminology

**Resource** — A dataset served at `/v1/{name}` with list, `meta`, and detail routes generated from one ResourceFactory config.

**Envelope** — JSON shape with `success`, optional `data` or `error`, and `meta` on list responses.

**Staging** — Import tables where rows are validated before promotion to `public`.

**Batch** — Identifier tying imported rows together for validation and migration jobs.

## FAQ

### Do I need an API key?

No for basic use. Keys raise rate limits and unlock future paid capabilities.

### Is there GraphQL?

Not currently—only REST under `/v1`.

### Where is the OpenAPI spec?

Generated from the API and published with the docs as the project matures; until then use `/meta` and each resource’s `/meta` route.

### How do I report bad data?

See [Contributing](/contributing). Include sources and the affected resource.
