## About the API

OpenNG groups public data under resource routes like `/v1/holidays`. Most work starts with discovery, then a resource `meta` endpoint, then the dataset itself.

<Cards className="grid gap-4 sm:grid-cols-2 xl:grid-cols-3">
  <Card title="Base URL" icon={<Compass />}>
    Production requests go to `https://api.openng.dev`. Local development usually runs at `http://localhost:3000`.
  </Card>
  <Card title="Discovery" icon={<Search />}>
    Use `GET /meta` to inspect available resources and versions before wiring a client.
  </Card>
  <Card title="Health" icon={<Activity />}>
    Use `GET /health` for readiness checks behind load balancers, monitors, and CI.
  </Card>
</Cards>

### Versioning

Paths are grouped as `/v1/{resource}`. Adding fields or filters is non-breaking. Incompatible changes to a specific resource earn a new `/v2/{resource}` only when needed.

## Create an API key

If you are building a script, backend integration, or worker, use an API key instead of browser cookies.

<Callout title="Store it once">

After you create a key in the dashboard, copy it immediately into a password manager, secret store, or environment variable. The plaintext token is shown once.

</Callout>

### Dashboard flow

1. Sign in on the main OpenNG app with your email.
2. Open `Dashboard -> API keys`.
3. Create a key and save the returned token.
4. Send it as a bearer token on API requests.

<CodeBlockTabs defaultValue="curl">
  <CodeBlockTabsList>
    <CodeBlockTabsTrigger value="curl">curl</CodeBlockTabsTrigger>
    <CodeBlockTabsTrigger value="fetch">fetch</CodeBlockTabsTrigger>
  </CodeBlockTabsList>
  <CodeBlockTab value="curl">

```bash
export OPENNG_API_KEY="ong_live_your_key_here"

curl -sS "https://api.openng.dev/v1/holidays?limit=5" \
  -H "Authorization: Bearer $OPENNG_API_KEY"
```

  </CodeBlockTab>
  <CodeBlockTab value="fetch">

```ts
const response = await fetch("https://api.openng.dev/v1/holidays?limit=5", {
  headers: {
    Authorization: `Bearer ${process.env.OPENNG_API_KEY}`,
  },
});

const json = await response.json();
```

  </CodeBlockTab>
</CodeBlockTabs>

### Rotation

Revoke the current key, then create a new one. Old keys stop working immediately once revoked.

## Authentication

Choose the mode that matches your client.

<Cards className="grid gap-4 sm:grid-cols-2 xl:grid-cols-3">
  <Card title="Browser sessions" icon={<LockKeyhole />}>
    The web and dashboard apps use passwordless email links and an `HttpOnly` session cookie.
  </Card>
  <Card title="API keys" icon={<KeyRound />}>
    Server-to-server and scripted access should use bearer tokens with the `ong_live_` prefix.
  </Card>
  <Card title="Anonymous access" icon={<ShieldCheck />}>
    Public `GET` routes work without a key, but limits are lower than authenticated usage.
  </Card>
</Cards>

### When to use what

- Use **browser sessions** when a human is signing into the app.
- Use **API keys** for scripts, cron jobs, workers, and backend services.
- Use **anonymous access** for quick experiments and early prototypes.

## Terminology

<Cards className="grid gap-4 sm:grid-cols-2">
  <Card title="Resource" icon={<Compass />}>
    A dataset served at `/v1/{name}` with list, `meta`, and detail routes generated from one config.
  </Card>
  <Card title="Envelope" icon={<Braces />}>
    The JSON response shape with `success`, optional `data` or `error`, and `meta`.
  </Card>
  <Card title="Staging" icon={<Activity />}>
    Import tables where rows are validated before promotion to `public`.
  </Card>
  <Card title="Batch" icon={<BookOpenCheck />}>
    The identifier tying imported rows together for validation and migration jobs.
  </Card>
</Cards>

## FAQ

<Accordions type="single">
  <Accordion title="Do I need an API key?" id="do-i-need-an-api-key">
    No for basic use. Keys raise rate limits and unlock higher-throughput access for scripts and integrations.
  </Accordion>
  <Accordion title="Is there GraphQL?" id="is-there-graphql">
    Not currently. OpenNG is a REST API under `/v1`.
  </Accordion>
  <Accordion title="Where is the OpenAPI spec?" id="where-is-the-openapi-spec">
    It will be published with the docs as generation matures. Until then, use `/meta` and each resource's `/meta` route for machine-readable discovery.
  </Accordion>
  <Accordion title="How do I report bad data?" id="how-do-i-report-bad-data">
    See [Contributing](/contributing) and include the official source, the affected endpoint, and the identifiers you received.
  </Accordion>
</Accordions>
