## Introduction

OpenNG is an open REST API for Nigerian public data. Data that currently lives in government PDFs, broken portals, and scattered Excel files is collected, cleaned, validated, and served through clean versioned endpoints.

OpenNG has two parts:

<Cards className="grid gap-4 sm:grid-cols-2">
  <Card title="The API" href="#about-the-api" icon={<Compass />}>
    `api.openng.dev` serves data across multiple domains: public holidays, fuel
    prices, postal codes, and more.
  </Card>
  <Card title="The Explorer" href="#create-an-api-key" icon={<KeyRound />}>
    `openng.dev` gives you an interactive data explorer to browse and filter
    data without writing code.
  </Card>
</Cards>

OpenNG is fully open source. The code, the database schema, the data pipeline, and the seed files are all on GitHub. If you find an error in the data or want to add a new resource, contributions are welcome.

## About the API

OpenNG groups public data under resource routes like `/v1/holidays`. Production traffic uses `https://api.openng.dev`; local development often uses `http://localhost:3000`. Use `GET /meta` to discover resources and versions, and `GET /health` for readiness checks.

### Versioning

Paths are grouped as `/v1/{resource}`. Adding fields or filters is non-breaking. Incompatible changes to a specific resource earn a new `/v2/{resource}` only when needed.

## Make Your First Request

No installation required. OpenNG is an HTTP API — call it from any language, environment, or tool that can make a request.

**Run** issues the request against **`http://localhost:3000`** (typical local API). Tabs are the same code blocks as the rest of the docs; nothing appears below until a response returns.

You should see JSON with every Nigerian public holiday declared for **2026**: dates, types, observance notes, and source links.

}

// g++ -std=c++17 main.cpp -lcurl -o openng_try && ./openng_try

#include <curl/curl.h>
#include <iostream>
#include <string>

static size_t writecb(char *ptr, size_t size, size_t nmemb, void *userdata) {
  auto *out = static_cast<std::string *>(userdata);
  out->append(ptr, size * nmemb);
  return size * nmemb;
}

  curl_easy_setopt(curl, CURLOPT_URL, url);
  curl_easy_setopt(curl, CURLOPT_HTTPHEADER, headers);
  curl_easy_setopt(curl, CURLOPT_WRITEFUNCTION, writecb);
  curl_easy_setopt(curl, CURLOPT_WRITEDATA, &body);
  CURLcode rc = curl_easy_perform(curl);
  curl_slist_free_all(headers);
  curl_easy_cleanup(curl);
  if (rc != CURLE_OK) return 1;
  std::cout << body;
  return 0;
}

## Create an API key

If you are building a script, backend integration, or worker, use an API key instead of browser cookies.

<Callout title="Store it once">

After you create a key in the dashboard, copy it immediately into a password manager, secret store, or environment variable. The plaintext token is shown once.

</Callout>

### Dashboard flow

1. Sign in on the main OpenNG app with your email.
2. Open `Dashboard -> API keys`.
3. Create a key and save the returned token.
4. Send it as a bearer token on API requests.

### Rotation

Revoke the current key, then create a new one. Old keys stop working immediately once revoked.

## What You Can Build

OpenNG is infrastructure. The data is the ingredient — what you build is the product.

A few things developers use OpenNG data for:

- Fintech apps that shift payment deadlines and reminders around Nigerian public holidays
- Logistics platforms that factor fuel prices by state into delivery estimates
- EdTech tools that look up school availability by LGA for placement
- Compliance tooling that validates registered NAFDAC products before stocking
- Address flows that use postal codes to standardise user input

If you ship something on OpenNG, [open a GitHub issue](https://github.com/stephcrown/openng/issues/new) with the **Built with OpenNG** label so we can track it.

## FAQ

<Accordions className="getting-started-faq" type="single">
  <Accordion title="Do I need an API key to use OpenNG?" id="faq-api-key">
    No. Anonymous requests are allowed and work immediately with no signup.
    Anonymous access is rate-limited to **500 requests per day**. For higher
    limits and access to bulk exports where available, create a free account at
    [openng.dev/dashboard](https://openng.dev/dashboard).
  </Accordion>
  <Accordion title="How often is the data updated?" id="faq-updates">
    It depends on the resource. Public holidays update when the Federal
    Government makes a new declaration — roughly 8–10 times per year. Fuel
    prices update monthly from NBS data. Static resources like postal codes
    update when the underlying data changes. Every API response includes a
    `last_updated` field, and `update_frequency` (where applicable) is described
    in each resource’s `/meta` response.
  </Accordion>
  <Accordion title="Is the data accurate?" id="faq-accurate">
    Every data point is traceable to its source: the `source_url` field on each
    record points to the original government publication. If you find an error,
    open a [GitHub issue](https://github.com/stephcrown/openng/issues/new) — we
    take data quality seriously and fix errors quickly.
  </Accordion>
  <Accordion
    title="Can I use this in a commercial product?"
    id="faq-commercial"
  >
    Yes. OpenNG is **MIT licensed**. The data comes from government sources and
    is in the public domain. There are no restrictions on commercial use beyond
    the license and normal API terms.
  </Accordion>
  <Accordion
    title="What happens when my rate limit is hit?"
    id="faq-rate-limit"
  >
    You receive **429 Too Many Requests** with a **`Retry-After`** header
    (seconds to wait). Successful responses include **`RateLimit-Limit`**,
    **`RateLimit-Remaining`**, and **`RateLimit-Reset`** so you can track usage
    before hitting the cap. Details in [Rate limits](/rate-limits).
  </Accordion>
  <Accordion
    title="A resource I need doesn't exist yet."
    id="faq-resource-request"
  >
    Open a [GitHub issue](https://github.com/stephcrown/openng/issues/new) with
    the **resource request** label: describe the data you need and what you
    would use it for. The roadmap is driven by real developer requests; if a
    resource is feasible and the data is publicly available, it gets built.
  </Accordion>
  <Accordion
    title="The data for a specific date looks wrong."
    id="faq-wrong-record"
  >
    Open a [GitHub issue](https://github.com/stephcrown/openng/issues/new) with
    the record **`id`** from the response, the value you received, what you
    believe is correct, and your source. We verify and correct as fast as we can
    — typically within **48 hours** for clear reports.
  </Accordion>
</Accordions>

The [GitHub repository](https://github.com/stephcrown/openng) tracks roadmap discussion and project updates.
