## Browser sessions

The marketing and dashboard apps use **passwordless email**. Links are single-use and short-lived; successful verification sets an **HttpOnly** session cookie.

## API keys

Server-to-server and script access should use **API keys** rather than scraping cookies. Keys are hashed in the database; plaintext never persists after creation.

## Anonymous access

Public `GET` endpoints work without a key at lower rate limits—ideal for experiments and prototypes.
