## Introduction

**OpenNG** makes Nigerian public data easy to consume: fuel prices, holidays, reference geography, and more—served through a consistent JSON API with pagination, filters, and clear documentation. Data is sourced from official releases and community pipelines, validated, and exposed under stable **`/v1`** paths.

OpenNG has several parts—use the cards below to jump in, or follow the sidebar for the full story.

### Terminology

**Resource:** A dataset exposed under `/v1/{name}` (for example `holidays`) with list, `meta`, detail, and export (where enabled).

**Envelope:** Every JSON response uses a shared shape: `success`, `data` or `error`, and `meta` on list routes.

**Staging vs public:** Imports land in `staging`, validation procedures run, then migration promotes rows to `public` tables the API reads.

**ResourceFactory:** Server-side config that generates list, meta, detail, and export routes from one definition—no one-off controllers for standard resources.

## First requests

A minimum of **Node.js 22** is recommended for local tooling; the API itself is consumed over HTTPS from any stack.

```bash title="Health"
curl -sS "https://api.openng.dev/health"
```

```bash title="List holidays (example)"
curl -sS "https://api.openng.dev/v1/holidays?year=2025&limit=5"
```

```bash title="Registry"
curl -sS "https://api.openng.dev/meta"
```

## FAQ

### Who is OpenNG for?

Developers building dashboards, research tools, and civic apps that need dependable Nigerian public data without scraping PDFs by hand.

### Do I need an API key?

No. Anonymous access works with stricter rate limits. Sign in on [openng.dev](https://openng.dev) to create a key when you need higher throughput.

### How often does data update?

It depends on the resource—see each resource’s **meta** response for `last_updated`, `freshness`, and `source_url`.

### Where do I report mistakes?

Use [Contributing](/contributing): file an issue or a PR against the relevant pipeline; we run validation before anything reaches production.

### Is there a GraphQL endpoint?

Not today—the API is REST-only, versioned per resource under `/v1`.

### For client apps

Use `https://api.openng.dev` as the base URL. From browsers, respect CORS: public `GET` under `/v1/*` is intended to work from documented web origins; authenticated flows use cookies on the main app.

### For scrapers and bulk jobs

Prefer an API key, backoff on `429`, and read `Rate-Limits` / `Retry-After` response headers.
