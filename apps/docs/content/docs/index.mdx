## Mission

OpenNG collects, validates, and publishes **Nigerian public data**—fuel, holidays, geography, and more—as **stable JSON** behind a single API surface. The goal is to replace brittle PDF scraping with something you can call from any stack and trust in production.

## Who uses OpenNG

### Product builders

#### Dashboards and internal tools

Teams wire OpenNG into **BI tools**, internal portals, and ops dashboards where fresh numbers beat static spreadsheets.

#### Research and journalism

Analysts and reporters use filters and exports to compare periods, states, and categories without maintaining one-off scrapers.

### Platform operators

If you run civic infrastructure, OpenNG is meant to sit beside your own services: **cache aggressively**, respect rate limits, and fall back to documented error codes when something drifts.

## How data reaches you

### Ingest

Imports land in **staging** with batch identifiers. Scrapers and manual pipelines write rows there first—never straight to the public tables the API reads.

### Validate

Stored procedures score each batch: required fields, referential integrity against `ref` tables, and domain rules (for example valid holiday calendars).

### Serve

Validated rows migrate to **`public`** schema tables. The API reads **only** from `public` and `ref`, so what you see in `/v1` matches what passed validation.

## Long-form reference (scroll test)

The sections below exist so you can **scroll** past the viewport and confirm the table of contents keeps up. They repeat concepts from other pages in more detail.

### Reliability

OpenNG is designed for **uptime-sensitive** clients: deterministic URLs, cache-friendly `GET`s, and explicit `429` responses when you exceed quotas. Prefer exponential backoff and jitter when you retry.

#### Caching hints

List endpoints emit pagination metadata so you can prefetch the next page without guessing sizes. Immutable reference rows (states, LGAs) should live in your own cache with long TTLs.

#### Idempotency

`GET` requests are safe to repeat. Do not expect idempotent `POST` on the public API—writes for contributed data go through separate flows documented on the main site.

### Observability

Correlate client timestamps with `meta.last_updated` on each response. When numbers look stale, check the advertised `source_url` and `freshness` before opening an issue.

#### Headers worth logging

Store anonymized samples of rate-limit headers to tune your worker pools. Never log bearer tokens or full API keys.

#### Tracing requests

When debugging with maintainers, include resource name, approximate time window, and a redacted example query string—no secrets.

### Geography joins

Most datasets eventually expose `state_id` or `lga_id`. Join against `/v1/ref/states` and `/v1/ref/lgas` **before** you aggregate, so you do not fork state names locally.

#### Slugs vs ids

Prefer **stable ids** in your database and human-readable **slugs** in URLs you share. The API exposes both where applicable.

#### FCT handling

The Federal Capital Territory is modeled like a state for filtering; LGAs under FCT behave like any other state’s LGAs.

### Versioning policy

Non-breaking additions—new optional filters, new columns in `data`—do not change the `/v1` path. Breaking removals or incompatible filter semantics require a new resource version and a migration period announced in the [Changelog](/changelog).

#### Deprecation headers

When a field or route is phased out, responses may include deprecation hints; watch the changelog and upgrade before hard removal.

#### Client libraries

Official SDKs may lag the API slightly—always trust the live `/meta` response when in doubt.

## FAQ

### Do I need an API key to try OpenNG?

No. Start anonymous, then add a key when you need higher limits or automation without throttling.

### Where is the OpenAPI document?

It will ship alongside the docs as generation matures. Until then, use `/meta` and each resource’s `/meta` route for machine-readable field and filter definitions.

### How do I report incorrect rows?

Use [Contributing](/contributing). Link to the official source you expect, the endpoint you called, and the identifiers you received.

### Can I use OpenNG from the browser?

Yes for public `GET`s where CORS allows your origin. For authenticated dashboard flows, use the main web app; for scripts, use API keys server-side.

### Is there a sandbox base URL?

Use a local deployment during development; production is `https://api.openng.dev`. There is no separate sandbox cluster yet.

### What about bulk exports?

Export routes are gated by tier and policy. Watch dataset pages for when `/export` opens up per resource.

Structured logging for each HTTP request; operators use them for latency and error analysis without leaking secrets.
