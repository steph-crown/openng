## Data corrections

Open an issue with sources, or submit a PR through the contribute flow on the main site when available.

## New resources

Pipeline: research → staging import → validation procedures → migration to `public`. Coordinate schema changes with maintainers early.

## Code

The API and scrapers live in the monorepo; follow project TypeScript and testing conventions.
