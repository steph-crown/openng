Delivery is tracked in the monorepo **roadmap** and issues. This page is a high-level pointer; the source of truth for checkboxed groups and ordering is the repository.

## Where to look

- **[ROADMAP.md](https://github.com/stephcrown/openng/blob/main/ROADMAP.md)** — grouped implementation plan for the platform and API
- **[GitHub issues](https://github.com/stephcrown/openng/issues)** — discussion, bugs, and smaller tasks

New **data resources** follow the pipeline described in the project docs: research, staging import, validation, migration to `public`, then `ResourceFactory` routes under `/v1`.

## Contributing direction

If you want a feature or dataset prioritized, open an issue with **use case**, **source**, and **expected consumers**. For data corrections, see [Contributing](/contributing).
